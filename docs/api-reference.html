    JsonSvcEx v0.1.0 API Reference

    Modules

    Controller.About

    Example About controller with sub enpoint route and failover 404 matcher all endpoints are aliased with /about/* based on Router.Base

    Controller.Account

    Account controller manages details of account modifications POST - request create of Account details PATCH - request modification of account details DELETE - request removal of Account details TODO: GET - get read optimized account details

    Controller.Root

    Root (/) and health endpoints to return service details and act as an external watchdog resource

    ErrorReporter.Honeybadger

    Reports errors to Honeybadger adds option to ignore report flow

    JsonSvc

    Base application supervisor that starts the service api router Router.Base

    KafkaHandlers.Account.RequestCreate

    KafkaHandlers for account controller Publishes to kafka topic using worker defined by @worker_id a random partition is selected per message

    KafkaHandlers.Workers

    Kafka Workers - create workers with atoms for handling different controller streams @workers list of worker modules that contain interface kafka_meta

    Model.Account.Request

    Account request object

    Response.Json

    Poison Encoder for handling json response bodies with status code sets default values for body=%{} and status=200 supports skipping one or both extra params

    Router.Base

    Base Router, handles all forwarding requests to separate controllers based on each controller’s defined path
